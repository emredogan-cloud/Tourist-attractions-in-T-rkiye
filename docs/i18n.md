# Internationalization (i18n) Workflow

## Supported locales

| Code | Native | Direction | Status |
|------|--------|-----------|--------|
| `tr` | Türkçe | LTR | Default. Source language for UI strings. |
| `en` | English | LTR | Hand-translated, complete. |
| `ar` | العربية | **RTL** | Phase 13. Skeleton complete. Cultural review required for photo captions before launch. |
| `ru` | Русский | LTR | Phase 13. Skeleton complete with proper plural rules (one/few/other). |
| `de` | Deutsch | LTR | Phase 13. Skeleton complete. |

## File layout

- UI strings: `src/messages/{locale}.json` — keyed identically across locales.
- Domain content (attraction translations): `AttractionTranslation` rows keyed by `(attractionId, locale)`. Currently only `tr` + `en` are populated; `ar` / `ru` / `de` fall back to `tr` automatically via the service layer's `pickTranslation`.

## RTL handling

`src/lib/i18n/config.ts` exports `isRtl(locale: Locale)`. The locale layout sets
`<html dir="rtl">` for `ar`. Tailwind utilities are direction-agnostic where
possible; check `text-balance`, `justify-end`, and any explicit `left-`/`right-`
positioning when adding new components — prefer `start-`/`end-` logical
properties.

## Crowdin (production workflow)

The repo is ready to wire up to Crowdin:

1. Create a Crowdin project; source language `tr`, target `en, ar, ru, de`.
2. Add a `crowdin.yml` at repo root pointing source files at
   `src/messages/tr.json` and translation files at `src/messages/%two_letters_code%.json`.
3. Configure GitHub integration → push commits whenever translations change.
4. CI guards: a `pnpm exec crowdin status` step would flag missing strings; for
   now, missing keys silently fall back to the value-as-key.

## Adding a new locale

1. Add the locale code to `src/lib/i18n/config.ts → locales` and `mvpLocales`.
2. Copy `src/messages/en.json` to `src/messages/<code>.json` and translate.
3. If RTL, add the locale to `rtlLocales`.
4. The sitemap and `hreflang` alternates are generated automatically from `mvpLocales`.

## Domain content rollout

Adding `ar`/`ru`/`de` translations for the curated attractions is the single
remaining piece for full parity with `tr`/`en`. The service layer handles
fallbacks gracefully — there is no engineering work needed beyond inserting
rows.
